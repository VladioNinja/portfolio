Leadership & Delivery

How I work with teams, risk and codebases.

Senior Unity work is not only implementation. The recurring pattern in my roles is owning unclear feature areas, making architecture decisions explicit, and keeping production delivery moving with art, design and engineering.

Feature ownership

Own Unity features from technical design through release and post-launch updates, including gameplay architecture, backend integration and platform-specific constraints.

Reusable systems

Favor state-driven, event-driven and ScriptableObject-based systems that can be reused across experiences without unnecessary rewrites.

Mentorship

Lead small Unity teams, run code reviews, set code quality standards and keep implementation choices understandable for other programmers.

Delivery clarity

Communicate progress, blockers, risks and trade-offs clearly across engineering, art and design so gameplay iteration stays fast and maintainable.
